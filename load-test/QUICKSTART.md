# Быстрый старт

## Самый простой способ (Node.js)

1. Откройте два терминала

2. В первом терминале запустите мониторинг:
```bash
npm run load-test:monitor
```

3. Во втором терминале запустите нагрузочный тест:
```bash
npm run load-test
```

Готово! Результаты будут выведены в консоль.

## С настройками

```bash
# Измените URL приложения
BASE_URL=http://localhost:3000 npm run load-test

# Измените количество пользователей
CONCURRENT_USERS=200 npm run load-test

# Измените длительность теста (5 минут = 300 секунд)
DURATION_SECONDS=300 npm run load-test
```

## Полный тест с мониторингом (одной командой)

```bash
cd load-test
./run-full-test.sh
```

## Что дальше?

1. Проверьте результаты в консоли
2. Откройте `load-test-resources.json` для детальной статистики ресурсов
3. Сравните метрики с ожидаемыми значениями (см. README.md)

## Типичные проблемы

**Высокое время ответа (>2s)**
→ Проверьте базу данных, индексы, количество соединений

**Много ошибок (>5%)**
→ Проверьте лимиты соединений, квоты API, логи ошибок

**Высокая нагрузка CPU (>80%)**
→ Увеличьте ресурсы сервера или оптимизируйте код

**Высокая нагрузка памяти (>85%)**
→ Проверьте на утечки памяти, оптимизируйте запросы
